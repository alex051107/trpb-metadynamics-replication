判断标准：ΔG(closed−open) 阈值
ΔG < −2 kcal/mol → 🟢 强活性候选（类 LBCA-TrpB，推荐湿实验）
ΔG = −2 到 0 → 🟡 中等活性（可能有活性，需优化）
ΔG = 0 到 +2 → 🟠 低活性（类 WT-PfTrpB，大概率无效）
ΔG > +2 → 🔴 无活性（COMM domain 无法关闭，直接丢弃）

2苏娜（Sílvia Osuna）在做什么——完整梳理

苏娜是谁
Sílvia Osuna，西班牙赫罗纳大学（Universitat de Girona）教授 + ICREA 研究员，CompBioLab 创始人。世界上最深入研究 TrpB 构象动力学的计算化学家。她的核心主张："酶的催化活性取决于它的构象系综（conformational ensemble），而不只是活性位点的静态结构。"
JACS 2019 解码 TrpS 进化中别构驱动的构象系综
苏娜做了什么

对三种 TrpB 变体（WT-PfTrpB、0B2-TrpB、LBCA-TrpB）运行 Well-Tempered MetaDynamics，计算 COMM domain 的二维自由能景观（2D FEL）。

核心发现
发现①： WT-PfTrpB（孤立，无活性）的 FEL 显示开放态深而稳，闭合态高出 3.5–5 kcal/mol。
发现②： LBCA-TrpB（6个突变，高活性）的 FEL 完全反转：ΔG(closed−open) = −2.3 kcal/mol，闭合态是全局极小值。
发现③： FEL 的 ΔG 和实验 kcat 之间存在定量相关性——可以用计算预测活性！
发现④： 6个突变都位于别构通信网络上，通过长程效应传递对 COMM 的稳定化。
与你的项目的连接

这篇论文建立了你的判断标准（ΔG 阈值），也证明了 MetaDynamics FEL 方法对评估 AI 设计蛋白有效。

📄 DOI: 10.1021/jacs.9b03646 · 已有完整标注版：JACS2019_Deep_Annotation.html
ACS Catal 2021 远端突变的计算识别与实验验证（SPM 方法）
苏娜做了什么

开发了 SPM（Shortest Path Map）方法：从 MD 轨迹计算残基间动态相关性矩阵 → 转化为图 → 用 Dijkstra 算法找最短路径 → 识别别构通信热点。

然后把 SPM 和祖先序列重建（ASR）结合，从 413 个残基缩减到 6 个候选突变（SPM6），设计出 ANC3-SPM6 TrpB 变体。

核心发现
结果： SPM6 变体（6个突变，5个远端，0个在 COMM domain 内）使 ANC3-TrpB 的 kcat 提升 7倍，相当于 3 轮定向进化（筛选 3080 个变体）的效果——但只测试了 1 个变体！
FEL 验证： SPM6 的 FEL 显示 COMM 闭合态稳定性提升（虽然未完全达到 LBCA 水平），与活性提升定量吻合。
FEL 的坐标轴（重要：这就是你问的 s 和 z！）

s 轴（进度轴）： 沿参考 open-to-closed 路径的进度（由 X 射线晶体结构生成），从 0（开放）到 1（闭合）。

z 轴（偏离轴）： 从参考路径的均方差（MSD）距离，反映偏离最优路径的程度。

这两个轴就是这篇论文 FEL 图中的 x 和 y 坐标——不同于 JACS 2019 用 RMSD + 距离，这里用了 PATH 集体变量。

📄 DOI: 10.1021/acscatal.1c03950 · 完全开放，已下载全文 · SPM 网页工具：spmosuna.com
Prot Sci 2022 tAF2-MD：用模板化 AlphaFold2 快速估计构象异质性
苏娜做了什么

意识到 MetaDynamics 计算量太大（每个蛋白需要 400–1600 ns 并行模拟，约 3–7 天 GPU），所以开发了快筛替代方案：

tAF2-MD 流程： ① 用 Reduced-MSA（32–64 条序列）+ 结构模板 运行 AlphaFold2，生成约 60 个结构 → ② 对每个结构跑 10 ns 普通 MD → ③ 合并轨迹，评估构象覆盖度。

结果： tAF2-MD 能在约 1/10–1/50 的计算量下，近似重现 MetaDynamics 的构象分布。可以作为 MetaDynamics 之前的"粗筛"步骤。
与你的项目的连接

当你有 50+ 个 AI 设计候选蛋白时，对每个都跑完整 MetaDynamics 不现实。先用 tAF2-MD 过滤，再对最有希望的（例如 top 5–10）做 MetaDynamics。

2025年 苏娜实验室目前在做什么

根据最新信息，苏娜实验室当前（2025–2026）的主要工作方向：

① SPM 通用化： 将 SPM 方法推广到更多酶系统（不仅限于 TrpB），提供网页服务器 spmosuna.com，让更多团队可以直接使用。

把 HTML 批注导入 Zotero 的三个可行方案

方案① 推荐：zotero-better-notes 插件（windingwind） GitHub: windingwind/zotero-better-notes
功能：在 Zotero 内部创建富文本笔记，支持 HTML 内容导入，可以把你的批注 HTML 转化为 Zotero 的笔记系统。
方法：① 安装插件 → ② 在 Zotero 中手动创建一个论文条目（对应 JACS 2019）→ ③ 打开 Better Notes 编辑器 → ④ 粘贴 HTML 内容或用模板导入。这是目前最活跃维护的 Zotero 笔记增强插件。
方案② 最简单：把 HTML 作为文件附件导入 在 Zotero 中找到或新建 JACS 2019 的论文条目 → 右键 → "Add Attachment" → "Attach Stored Copy of File" → 选择 JACS2019_Deep_Annotation.html。
效果：HTML 文件直接存储在 Zotero 中，可以从 Zotero 打开并查看所有批注。格式完全保留。这是最可靠的方案，零设置。
方案③ 进阶：SingleFile 浏览器扩展 + Zotero GitHub: gildas-lormeau/SingleFile
SingleFile 可以将完整的 HTML 页面（包含样式和内嵌资源）保存为单一文件。如果你的 HTML 批注打开后依赖 Google Fonts CDN，用 SingleFile 可以打包成完全离线的单文件，再导入 Zotero。
关于 HTML 批注 → PDF → Zotero（更高级） 如果你想让批注在 Zotero 的 PDF 阅读器里显示，可以：浏览器打开 HTML → 打印 → 保存为 PDF → 导入 Zotero。但这样会丢失交互性（中文注释还在，但不能折叠/展开）。适合分享给别人。
