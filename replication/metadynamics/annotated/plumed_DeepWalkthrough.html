TL;DR — 这个文件干什么 + 为什么这样写

三行 directive 完成 WT-MetaD on a 2D path CV：PATHMSD 把 112 个 Cα 的瞬时构象映射到 (s, z) ∈ ℝ² 上的两个标量；METAD 在 (s, z) 平面上以 1 ps 节奏沉积自适应 Gaussian 偏置势；PRINT 每 1 ps 把 (s, z, bias) 写入 COLVAR 供后续诊断。文件本体只有 3 行可执行；其余 64 行注释是 6 个独立 bug 的封存现场 (FP-018 / 020 / 022 / 023 / 024 / 025) 与每个数值参数的 Source 出处链——这是 PI/reviewer 检查 "数字哪来" 的唯一可信入口。

设计哲学（先看这段再读细节）

DESIGN

这个文件采用 "3 行 directive + 厚注释" 结构。Directive 行是给 PLUMED 解析的（机器读），注释是给后来人读的（人读）。

每个数值都有 Source 标签（SI-quote / SI-derived / Our-choice / PLUMED-default / Legacy-bug），且每个 Our-choice 数值都标了 物理范围 + ×2/÷2 影响。

每个曾经踩过的坑都封存在 FP-XXX 注释块里——不是给读者讲故事，是为了让下一个想"删掉过时注释"的人看到代价。

结构功能：把 "可执行内容" 和 "为什么这样写" 物理分离，但保留在同一个文件里。

逻辑角色：self-documenting artifact

💡 核心约束：annotated 版和 production 版必须 byte-identical（剥掉注释和空行后）。这通过 README.md 里的 diff 命令强制验证。任何破坏这个约束的编辑都不能 commit。

对项目的意义：当 PI 或 reviewer 问 "你这个 LAMBDA 哪来的"，文件本身就能回答；不需要切到 Slack 历史或 git blame。

P2
#   FP-018 (2026-04-04): Å⁻² → nm⁻² × 100 unit conversion added for LAMBDA

第一个被封存的 bug：LAMBDA 的单位换算。SI 给的 MSD 在 Å²，PLUMED 内部用 nm²，所以 λ = 2.3 / MSD 在两套单位下差了 100×（Å→nm 是 10×，MSD 是平方所以 100×）。

逻辑角色：FP-history entry #1

⚠️ 陷阱：PLUMED 文档里的公式既出现过 nm 也出现过 Å 写法，看不同 tutorial 时混淆是常见来源。
🔗 对未来：任何"换 path / 换原子集" 都要重新跑 generate_path_cv.py，新 LAMBDA 会自动按 nm² 输出，但仍要肉眼复核数量级。

P4
#   FP-022 (2026-04-08): FUNCPATHMSD workaround used per-atom-mean RMSD
#     semantics, so LAMBDA had to be recomputed from per-atom MSD (379.77)
#     instead of the original total-SD value (3.391). This was a silent
#     N_atoms = 112× error in the FUNCPATHMSD path. Now moot — we are
#     switching back to PATHMSD.

这是 sister bug：FUNCPATHMSD 内部的 RMSD action 输出的是 per-atom mean RMSD，而我们最初按 total SD 算 LAMBDA。结果差了 112×（恰好等于原子数）。这个 bug 是 silent 的——程序不报错，但 FES 形状是错的。

段落功能：说明 "切回 PATHMSD 之前的最后一个隐患"——并强调这种 silent bug 比报错更危险。

逻辑角色：FP-history entry #3 (silent numerical bug)

⚠️ 致命陷阱：不同 PLUMED action 对 "MSD" 的定义不一样！
  • RMSD action: per-atom mean，单位 length²
  • PATHMSD action: per-atom mean MSD，单位 length²
  • 原始 Branduardi 公式可能写 total SD（看文献版本）
所以 LAMBDA = 2.3 / MSD 这个公式必须确认 MSD 用的是哪种 convention。
🔗 对项目：FP-022 之后，所有 generate_path_cv.py 函数都加了 assertion：calculate_msd 必须 docstring 明示 "per-atom mean MSD in Å²"。函数名和返回值语义一致是从这里来的。

#   FP-024 (2026-04-15): SIGMA=0.05 with ADAPTIVE=GEOM collapsed to adaptive
#     widths of 0.011–0.072 s-units (≤0.5% of path axis 1-15). Job 42679152
#     ran 50 ns, deposited 25000 Gaussians and accumulated 48 kJ/mol of bias,

这是项目最近一次重大灾难：50 ns 模拟跑完后，walker 完全没离开 Open basin。事后分析发现 SIGMA=0.05（单一 Cartesian）经 ADAPTIVE=GEOM 反投影到 s 方向后塌成 0.011 s-unit，远小于 path 1-15 的尺度。25000 个针尖 Gaussian 全部堆在 s≈1 处，bias 累到 12 kcal/mol 也推不出 basin。

段落功能：记录 "为什么现在的 SIGMA 行有 SIGMA_MIN/SIGMA_MAX"。

逻辑角色：FP-history entry #5 (silent sampling failure)

量值

累积 bias48 kJ/mol = 11.5 kcal/mol
walker 范围 s1.0–1.6 (path 总范围 1–15)

⚠️ silent 失败模式：HILLS 文件正常增长，COLVAR 正常输出，bias 正常累积——程序"看起来在工作"，但物理上没采样。这种失败比报错更难发现：必须画 s(t) 时间序列才能看出。

Step 1: PATHMSD action（P11–P14）—— 把 39268 原子塌成 (s, z) 两个数

P11 · Action 行
path: PATHMSD REFERENCE=path_gromacs.pdb LAMBDA=379.77

这是文件里第一行可执行 directive。PATHMSD 是 PLUMED 的内置 action（Branduardi, Gervasio, Parrinello 2007），把当前构象映射到沿参考路径的进度 s(R) 和偏离距离 z(R)。path 是 label，后续可用 path.sss 和 path.zzz 引用。

段落功能：整个文件最核心的一行——把高维构象空间塌成 2D。

逻辑角色：dimensionality reduction

🛠️ 底层公式（Branduardi 2007）：
s(R) = Σᵢ i·exp(−λ·MSDᵢ) / Σᵢ exp(−λ·MSDᵢ)
z(R) = −(1/λ)·log Σᵢ exp(−λ·MSDᵢ)
其中 MSDᵢ 是当前构象与参考帧 i 的 per-atom mean squared displacement（先 Kabsch 叠合）。
量含义
path.sss路径进度 (1=Open, 15=Closed)
path.zzz偏离路径距离 (nm²)
label "path"用户起的名字，后面 METAD 通过 path.sss 引用
🔗 对项目：这一行是整个 MetaD 复刻的物理基础。如果换成 contact-based CV 或 distance-based CV，整个 pipeline 都要重新设计。
P12 · REFERENCE 文件
REFERENCE=path_gromacs.pdb

这是 PATHMSD 的输入文件——一个 multi-model PDB，15 个 MODEL/ENDMDL 块。每帧 112 个 Cα 原子（COMM domain 88 + base 24）。Frame 1 = Open（来自 1WDW），Frame 15 = Closed（来自 3CEP），中间 13 帧线性插值。

段落功能：定义"参考路径"——MetaD 走的不是任意构象，而是这条预定义路径。

逻辑角色：reference geometry

属性值
帧数15
每帧原子数112 Cα (88 COMM + 24 base)
坐标单位Å (PDB 惯例) → PLUMED 自动转 nm
路径生成Cα 空间线性插值 (generate_path_cv.py)
endpoint 11WDW (Open)
endpoint 153CEP (Closed)
文件尾必须 ENDMDL，不能 END (FP-023)
⚠️ 路径质量决定 MetaD 质量：如果 path 不是真实物理路径（例如 endpoint 选错或线性插值跨过非物理构象），MetaD 会沿"虚假"路径采样，得到错误的 FES。
🔗 对项目：未来想用 SE(3)-Diffusion / STAR-MD-style 模型，关键就是替换这步——不再用线性插值生成 path，而是模型直接给出 path 上的中间构象分布。
P13 · LAMBDA 参数
LAMBDA=379.77

PATHMSD 公式中 exp(−λ·MSD) 的衰减系数。物理含义：邻帧权重 ≈ exp(−λ·⟨MSD⟩) ≈ 0.10（PLUMED 推荐值）→ 解出 λ = 2.3 / ⟨MSD⟩。

段落功能：控制 path CV 的"分辨率 vs 平滑性"trade-off。

逻辑角色：discretization parameter

SI-derived本地从 generate_path_cv.py 推出（SI 给公式不给数值）

×2 (759.54)邻帧权重 exp(−4.6) = 0.010 → walker 几乎只看最近一帧 → s(R) 变成阶跃函数 → 力传不到原子 → 卡帧
÷2 (189.89)权重 exp(−1.15) = 0.317 → 邻帧贡献过多 → s(R) 被平滑模糊 → FES 沿 path 丢分辨率
推荐区间2.3 / ⟨MSD⟩ 附近 ±50% (PLUMED-NEST best practice)
⚠️ FP-027 警告：SI p.S3 句尾的 "80" 是 total SD 单位 Å²，不是 λ。曾经误读成 λ=80 nm⁻² 花 3+ 小时错方向诊断。
🔗 对项目：更换 path（例如换成 16 帧或非线性插值）后必须重算，不能复用 379.77。
P14 · 输出字段命名
path.sss — 进度 (PATHMSD 命名)
path.zzz — 距离 (PATHMSD 命名)

vs.
path.s   — 进度 (FUNCPATHMSD 命名)
path.z   — 距离 (FUNCPATHMSD 命名)

这是 PATHMSD 和 FUNCPATHMSD 的接口差异。从 FUNCPATHMSD 切到 PATHMSD（FP-020 修复时）必须同时改 METAD 的 ARG，否则会报 "ARG not found"。

段落功能：提醒读者 "label 之后的字段名不能改，PLUMED 内部硬编码"。

逻辑角色：API contract

⚠️ 易错点：label 是用户随便起的（这里是 "path"），但字段后缀（.sss/.zzz 或 .s/.z）由 action 类型决定，改不了。FUNCPATHMSD ↔ PATHMSD 切换时必须同步改所有引用。
🔗 对项目：未来如果再换 CV action（例如换 PATHCV），同样的 grep 检查 ARG 行；目前只 1 处 (METAD)。

Step 2: METAD action（P15–P25）—— Well-Tempered 偏置势

P15 · Action 行
metad: METAD ARG=path.sss,path.zzz SIGMA=0.1 ADAPTIVE=GEOM SIGMA_MIN=0.3,0.005 SIGMA_MAX=1.0,0.05 HEIGHT=0.628 PACE=1000 BIASFACTOR=10 TEMP=350 FILE=HILLS

这一行 8 个数值参数，每个都决定 MetaD 的不同行为侧面。Well-tempered MetaD (Barducci 2008) 与原版区别：hill 高度按 exp(−V/(k_B·ΔT)) 自动衰减，保证最终 V(s) → −(1−1/γ)·F(s) 收敛。

段落功能：本文件最复杂的一行——8 参数 × 各自的 Source × 各自的物理含义 × 各自的 ×2/÷2 影响 = 项目 review 的主战场。

逻辑角色：main bias driver

🛠️ WT-MetaD 公式：
V(s, t+τ) = V(s, t) + W·exp(−V(s, t) / (k_B·ΔT))·G(s − s(t))
γ = (T+ΔT) / T = BIASFACTOR
最终: V(s) → −(1 − 1/γ)·F(s)
🔗 对项目：下面 P17–P25 逐个参数详解；每个都给 ×2 / ÷2 影响表格，方便 PI 评估"如果改这个会怎样"。
P16 · ARG / ADAPTIVE
ARG=path.sss,path.zzz
ADAPTIVE=GEOM

ARG：bias 施加在哪两个 CV 上 → 二维 MetaD。Gaussian hill 是椭圆。
ADAPTIVE=GEOM：自适应 sigma 模式，按 CV 空间局部几何调整 Gaussian 宽度（CV 变化快的地方用窄 hill，慢的地方用宽 hill）。SI 明示用 GEOM。

段落功能：声明"在 (s, z) 空间做 2D MetaD"+"用 PLUMED 自适应宽度"。

逻辑角色：CV space declaration

选项含义
ADAPTIVE=NONE (默认)固定 SIGMA，不自适应
ADAPTIVE=GEOM按 CV 几何变化自适应（SI 用这个）
ADAPTIVE=DIFF按 CV 时间涨落自适应
⚠️ FP-024 根因：ADAPTIVE=GEOM 下 SIGMA 含义从 "per-CV 宽度" 变成 "Cartesian 几何种子"——这个语义切换是 SIGMA 塌缩 bug 的源头。必须同时给 SIGMA_MIN/MAX 才安全。
🔗 对项目：如果未来想换 ADAPTIVE=DIFF，必须重新评估 SIGMA_MIN/MAX 的合理范围（DIFF 模式下 sigma 来自时间涨落，不是几何）。
P17 · SIGMA
SIGMA=0.1  (nm, Cartesian 几何种子)

ADAPTIVE=GEOM 模式下，SIGMA 是单一标量 Cartesian 长度（不是 per-CV 宽度）。PLUMED 反投影到 s 和 z 方向，得到两个 per-CV sigma，再被 SIGMA_MIN/MAX clip。

段落功能：给 PLUMED 一个"几何种子"，让自适应算法有起始尺度。

逻辑角色：geometric seed

Our-choiceSI 全文无数值 SIGMA（FP-025 警告假引用！）

历史：Pre-2026-04-15 = 0.05 → FP-024 崩塌 → 现在 0.1 + 强制下限。

×2 (0.2 nm)反投影后 per-CV sigma 翻倍 → Gaussian 更胖
+ 起始 basin 迅速被填平 → 更快出 O basin
− FES 分辨率下降，相邻 basin 可能糊成一个
÷2 (0.05 nm)回到 FP-024 灾难场景；sigma_s 反投影塌到 0.011

− bias 累到 12 kcal/mol 也推不出 O basin（Job 42679152 实测）

P20 · SIGMA_MAX[0] (s 方向)
SIGMA_MAX=1.0,0.05  (CV 单位)

Gaussian 在 path 进度 s 方向的最大宽度上限。防 ADAPTIVE 反投影出过大 sigma 把整条 path 糊成一坨。

段落功能：防 Gaussian 过胖把相邻 basin 合并。

− FES 分辨率差，O/PC/C 三态分不清

P21 · SIGMA_MAX[1] (z 方向)
SIGMA_MAX=1.0,0.05  (nm²)

z 方向上限。基于 Job 42679152 z 分布上半部。

段落功能：z 向 Gaussian 不能糊掉 on-path / off-path 区分。

P22 · HEIGHT
HEIGHT=0.628  (kJ/mol)

每个 Gaussian hill 的初始高度。WT-MetaD 下实际高度按 exp(−V/(k_B·ΔT)) 衰减，0.628 kJ/mol = 0.15 kcal/mol（SI 原文）。

段落功能：控制 bias 累积速度。

逻辑角色：bias deposition rate (per hill)

SI-quoteSI p.S3 "Initial Gaussian potentials of height 0.15 kcal/mol"

🛠️ 换算：0.15 kcal/mol × 4.184 = 0.6276 ≈ 0.628 kJ/mol
×2 (1.256)bias 积累速度 2×
+ 更快填平 FES basin
− 过量沉积 → 冲出物理合理区域 (z>>0.1) 概率↑；WT 收敛变慢
÷2 (0.314)bias 积累速度 0.5×
+ 更精细的 FES（适合收敛阶段，SI 也可能后期降 height）
− 50 ns 可能填不满 basin，walker 仍卡在起始态
推荐区间0.1–1.0 kJ/mol (Barducci 2008 WT-MetaD 原文建议 k_B·T 量级)
P23 · PACE
PACE=1000  (steps; dt=2 fs → 每 2 ps 一个 hill)

每多少 MD 步沉积一个 hill。PACE × dt = 2 ps（SI 明示）。直接决定 bias 累积的时间密度。

段落功能：控制 deposition 节奏。

逻辑角色：temporal cadence

SI-quoteSI p.S3 "hills ... deposited every 2 ps of MD simulation at 350 K"

×2 (2000 steps, 4 ps)deposition rate 0.5×
+ CV 有更多时间在两次 deposit 间松弛 → WT 收敛质量↑
− 填平同样 basin 需要 2× 真实时间 (50 ns → 100 ns)
÷2 (500 steps, 1 ps)deposition rate 2×
− CV 来不及松弛 → "nonadiabatic" 污染 → FES 估计有偏
不建议偏离 SI 值
推荐区间SI 明示 1000；偏离必须说明（例 1-walker 首探可提到 500 加速初扫）
P24 · BIASFACTOR
BIASFACTOR=10

Well-tempered 温度比 γ = (T+ΔT)/T。CV 空间有效采样温度 = γ·T = 3500 K（原子运动仍是 350 K）。决定 bias 衰减速度和最终 FES 回放系数 F(s) = −γ/(γ−1)·V_bias(s)。

段落功能：定义 WT-MetaD 的"温度梯度"——CV 走得多快、bias 衰减多快、最终 FES 怎么回放。

逻辑角色：WT temperature ratio

SI-quoteSI p.S3 "well-tempered adaptative bias with a bias factor of 10"

×2 (γ=20)T_eff = 7000 K (CV)
+ 采样能量更高的 barrier，能量阈值 ≈ γ·k_B·T
− WT 收敛时间 ↑（deposit 高度衰减更慢）；CV 漂远非物理区
÷2 (γ=5)T_eff = 1750 K
+ 更快收敛；bias 更快衰减到 0
− 采不到 > 5·k_B·T ≈ 3.5 kcal/mol 的 barrier
− 对 TrpB O→C barrier ~5–7 kcal/mol 可能采不全
推荐区间5–20 (Barducci 2008 原文区间)；SI 明示 10
P25 · TEMP
TEMP=350  (K)

MetaD bias 的物理温度。必须与 metad.mdp 的 ref_t 一致。物理依据：P. furiosus 是 hyperthermophile，生长温度 75–100°C。

段落功能：把 PLUMED 的统计力学温度和 GROMACS 的热浴温度绑定。

逻辑角色：physical temperature

SI-quoteSI p.S3 "2 ps of MD simulation at 350 K"

×2 (700 K)蛋白质可能已经 unfold → 整个模拟失效
÷2 (175 K)低于水的冰点；TIP3P 在此温度下非物理
推荐区间SI 明示 350；E. coli-native 蛋白会用 300/310 K
⚠️ 不要乱调：TEMP 必须匹配实验温度或生理温度。这是物理约束，不是优化参数。
🔗 对项目：检查清单：每次改 metad.mdp 的 ref_t 都要同步检查 plumed.dat 的 TEMP（grep 双向验证）。

Step 3: PRINT action（P26）—— 实时监测

P26 · PRINT 监测
PRINT ARG=path.sss,path.zzz,metad.bias FILE=COLVAR STRIDE=500

每 500 步（dt=0.002 ps × 500 = 每 1 ps）输出一行。MetaD 运行期间，COLVAR 是唯一能"看到系统在做什么"的方式（HILLS 只记录 deposit 事件，COLVAR 是连续时间序列）。

段落功能：可观察性 layer——把模拟内部状态导出供后续诊断/可视化。

逻辑角色：observability hook

列含义
timeps
path.sss当前进度 (1..15)
path.zzz当前距离 (nm²)
metad.bias当前点累积 bias (kJ/mol)
🛠️ 最快诊断： 
python3 -c "
import numpy as np; d=np.loadtxt('COLVAR',comments='#')
s=d[:,1]
print(f'O(s<5): {(s<5).sum()}  PC(5-10): {((s>5)&(s<10)).sum()}  C(s>10): {(s>10).sum()}')"
💡 STRIDE vs PACE：STRIDE=500 (1 ps) 比 PACE=1000 (2 ps) 更频繁——因为 HILLS 是 deposit 事件（不需要密），COLVAR 是监测数据（要看清涨落）。两者时间分辨率独立。
🔗 对项目：FP-024 silent failure 的关键诊断就是画 s(t) 时间序列。如果当时没有 COLVAR，可能 50 ns 跑完都不知道 walker 没动。

§A 参数依赖链 — 改一个参数会带动哪些

这张图回答 PI 常问的问题："如果我改 X，还要改什么？"

[换 path geometry (例 16 帧 / 不同 endpoint)]
   │
   ├─→ path_gromacs.pdb 改 (重跑 generate_path_cv.py)
   ├─→ ⟨MSD_adjacent⟩ 变 → LAMBDA 重算 (P13)
   ├─→ path 轴长变 → SIGMA_MIN[s] 按 2% 重算 (P18)
   └─→ path 轴长变 → SIGMA_MAX[s] 按 7% 重算 (P20)

[换原子集 (例 加 PLP heavy atoms)]
   │
   ├─→ path_gromacs.pdb 重生成
   ├─→ N_atoms 变 → ⟨MSD⟩ per-atom 变 → LAMBDA 重算
   └─→ FUNCPATHMSD ↔ PATHMSD 切换：必须改 ARG=path.sss,path.zzz
        ⇆ ARG=path.s,path.z (P14)

[换温度 (例 300 K 适配 E.coli)]
   │
   ├─→ metad.mdp ref_t = 300
   ├─→ plumed.dat TEMP=300 (P25)
   └─→ HEIGHT 单位换算重核 (kcal/mol → kJ/mol 不变，但 k_B·T 量级变)

[换 ADAPTIVE 模式 (GEOM → DIFF)]
   │
   ├─→ SIGMA 含义变 (Cartesian → CV space)
   ├─→ SIGMA_MIN/MAX 阈值重新评估
   └─→ 必须重跑 1-2 ns toy 验证 sigma 分布合理

[换 BIASFACTOR (γ)]
   │
   ├─→ FES 回放系数变: F(s) = −γ/(γ−1)·V(s)
   ├─→ 收敛时间估计变 (γ↑ → 慢)
   └─→ 可达 barrier 阈值变 (~γ·k_B·T)
