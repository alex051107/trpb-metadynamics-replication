# 阶段 3：Janitor（卫生员）

## 功能

标准化工作目录：规范文件路径、组织输入输出、准备 campaign 目录结构。

## 何时使用

- 项目文件分散、路径不清楚
- 输入/输出位置有歧义
- 文件名不一致
- 需要为本地或 HPC 环境准备可重现的工作区
- 原始资源和生成的工件混在一起

## 输入

无特定输入格式。检查现有目录结构和文件布局。

## 输出

选择以下之一：
- 清理后的目录方案说明
- 明确的输入/输出位置清单
- normalized filenames（标准化文件名）
- 变更日志

## 关键规则

1. **分离原始和生成**：raw inputs ≠ generated outputs
2. **使用相对路径**：优先项目内相对路径，而非绝对路径
3. **记录变更**：每次移动或重命名都要在笔记中记录
4. **不改内容**：只整理目录结构，不修改科学内容
5. **按活动分类**：benchmark / comparison / export 工件应该分开

## 工作流程

1. 检查现有 repro-workflow/ 目录结构
2. 规划 campaign 级目录：
   - `inputs/`（输入文件）
   - `configs/`（配置文件）
   - `runs/`（运行输出）
   - `analysis/`（分析结果）
   - `exports/`（导出产品）
   - `reports/`（报告）
3. 规范文件名
4. 在日志中记录每次变更

## 非目标

- 不要重新解释协议参数
- 如果外部资源仍然缺失，不要声称工作区"已就绪"
