# TrpB MetaDynamics 复刻：完整操作流程指南

## 文档说明

本文档整合了以下所有信息源，形成一份可复现的端到端操作手册：

- JACS 2019 主文 (Maria-Solano et al., JACS 2019, 141, 13049–13056, DOI: 10.1021/jacs.9b03646)
- JACS 2019 Supporting Information (ja9b03646_si_001.pdf, p.S1–S17)
- 项目仓库已有脚本 (`parameterize_plp.sh`, `heat*.in`, `plumed_trpb_metad_single.dat`)
- Longleaf HPC 环境实测经验

适用体系：PfTrpS / PfTrpB / PfTrpB⁰ᴮ² 在 Ain、Aex1、A-A、Q₂ 四种反应中间体下的 COMM domain MetaDynamics。

**当前 benchmark 目标**：PfTrpS(Ain) — 即天然复合体在 internal aldimine 状态。

---

## 全流程概览

```
PDB 下载 → PLP 参数化 (GAFF + RESP) → tleap 体系搭建
    → 最小化 (两步) → 阶梯加热 (0→350K, 7步) → NPT 平衡 (2ns)
    → NVT Production MD (500ns) → AMBER→GROMACS 转换 (ParmEd)
    → Path CV 构建 (15帧) → PLUMED 配置
    → WT-MetaDynamics (10 walkers) → FES 重建 (sum_hills)
    → 分析 (basin 识别 + productive closure 验证)
```

## 第二步：PLP 参数化 — GAFF + RESP（★ 当前 Pipeline Blocker）

> **来源**：JACS 2019 SI p.S2 — "MD simulation parameters for the reaction intermediates (IGP, G3P, Ain, Aex1, A-A, Q₂) were generated with the antechamber module of AMBER16 using the general amber force-field (GAFF) with partial charges set to fit the electrostatic potential generated at the HF/6-31G(d) level with the RESP model. These charges were calculated using the Gaussian09 software package."

### 2.1 为什么需要参数化

标准蛋白力场 (ff14SB) 只认识 20 种标准氨基酸。PLP 与 Lys82 形成共价 Schiff base (E(Ain)/LLP)，ff14SB 完全不认识。需要补充三类信息：

| 需要补齐 | 工具                        | 作用                              |
| -------- | --------------------------- | --------------------------------- |
| 原子类型 | `antechamber -at gaff`    | 决定 bonded/nonbonded 参数模板    |
| 部分电荷 | `RESP` on Gaussian output | 决定静电相互作用                  |
| 缺失参数 | `parmchk2`                | 补齐 GAFF 库中未覆盖的键角/二面角 |

### 2.2 关键参数总结

| 参数                    | 值                          | 来源                                       | 是否需要改     |
| ----------------------- | --------------------------- | ------------------------------------------ | -------------- |
| 力场                    | GAFF (不是 GAFF2)           | SI p.S2 "general amber force-field (GAFF)" | **不变** |
| 电荷方法                | RESP (不是 AM1-BCC)         | SI p.S2 "RESP model"                       | **不变** |
| QM level                | HF/6-31G(d)                 | SI p.S2 "HF/6-31G(d)"                      | **不变** |
| QM software             | Gaussian (SI用09, 我们用16) | SI p.S2 "Gaussian09"                       | 版本可更新     |
| 总电荷 (Ain/LLP capped) | -2                          | 化学推导 + 项目验证                        | 换中间体要重算 |
| 多重度                  | 1 (闭壳)                    | 标准                                       | **不变** |

> **⚠️ FP-009 教训**：Osuna 2019 明确用 RESP，不是 AM1-BCC。BCC 更快但对 PLP 这种带磷酸基、共轭体系的分子不够准确。

### 2.3 Capped Fragment 提取

> **为什么要 capping**：LLP 是 polymer-linked residue（和蛋白主链共价相连）。断键边界如果不补，QM 电荷分布会失真。ACE-LLP-NME 让 RESP 看到更合理的局部电子环境。

```bash
# 从 5DVZ 提取 LLP 残基 + 相邻主链原子
# 加上 ACE (N-端 cap) 和 NME (C-端 cap)

# Step 1: 提取 LLP 残基（chain A, residue 82）
grep "^ATOM.*A  82\|^HETATM.*LLP.*A  82" 5DVZ.pdb > llp_raw.pdb

# Step 2: 手动添加 ACE 和 NME caps
# ACE: C, O, CH3 接在 N-端
# NME: N, H, CH3 接在 C-端
# 详见 parameterize_plp.sh Section 3

# Sanity checks:
# - Capped fragment 应有 ~54 atoms (24 heavy + H + caps)
# - 总电荷 = -2
# - 多重度 = 1
```

### 2.4 Gaussian 单点 + ESP 计算

```bash
# Gaussian 16 input file template
cat > llp_resp.gjf << 'EOF'
%mem=8GB
%nproc=8
%chk=llp_resp.chk
# HF/6-31G(d) SCF=Tight Pop=MK IOp(6/33=2,6/42=6)

ACE-LLP-NME RESP charges for TrpB Ain intermediate

-2 1
[atom coordinates here]

EOF

# 提交到 Longleaf
# module load gaussian/16c02
# g16 llp_resp.gjf
```

| Gaussian 关键字        | 含义                        | 来源                                                                                                                     |
| ---------------------- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `HF/6-31G(d)`        | Hartree-Fock, 6-31G(d) 基组 | SI p.S2                                                                                                                  |
| `Pop=MK`             | Merz-Kollman ESP fitting    | RESP 需要                                                                                                                |
| `IOp(6/33=2,6/42=6)` | 控制 ESP 输出精度           | RESP 工作流标准。**注意**：不要加 `IOp(6/50=1)`，它在 Gaussian 16 中会导致 "Blank file name read" 错误（FP-013） |

### 2.5 RESP 电荷拟合 + 力场参数生成

```bash
# Step 1: antechamber — 从 Gaussian log 提取 RESP 电荷 + 分配 GAFF 原子类型
# 注意：用 -fi gout 读 Gaussian log，不要用 -fi gesp（FP-013/FP-014）
antechamber -i llp_resp.log -fi gout \
            -o LLP.mol2 -fo mol2 \
            -c resp -at gaff \
            -nc -2 -m 1 \
            -rn LLP

# Step 2: parmchk2 — 检查 GAFF 库中是否有缺失参数，生成 frcmod
parmchk2 -i LLP.mol2 -f mol2 -o LLP.frcmod -a Y

# Sanity checks:
# - LLP.mol2 中所有原子应有非零电荷
# - 电荷总和应 ≈ -2.000
# - LLP.frcmod 中 ATTN 标记的参数需要人工检查合理性
```

> **⚠️ FP-010 教训**：对无氢 PDB 直接跑 antechamber 会把芳香体系类型分错（如吡啶环）。务必先加氢。

## 第三步：tleap 体系搭建

> **来源**：JACS 2019 SI p.S2 —
> "Each enzyme system was immersed in a pre-equilibrated cubic box with a 10-Angstrom buffer of TIP3P water molecules, resulting in the addition of approximately 15,000 water molecules. Afterwards, the systems were neutralized by the addition of explicit counterions (Na⁺ or Cl⁻). All subsequent calculations were done using a modification of the amber99 force field (ff14SB)."

### 3.1 力场组合

| 组件         | 选择                    | 原文依据                                                       |
| ------------ | ----------------------- | -------------------------------------------------------------- |
| 蛋白         | `ff14SB`              | "a modification of the amber99 force field (ff14SB)" (SI p.S2) |
| PLP/cofactor | `GAFF` + RESP charges | "the general amber force-field (GAFF)" (SI p.S2)               |
| 水           | `TIP3P`               | "TIP3P water molecules" (SI p.S2)                              |
| 离子         | Na⁺ / Cl⁻             | "explicit counterions (Na⁺ or Cl⁻)" (SI p.S2)                |

### 3.2 tleap 输入

```bash
cat > tleap.in << 'EOF'
source leaprc.protein.ff14SB
source leaprc.gaff
source leaprc.water.tip3p

# Load PLP parameters
loadAmberParams LLP.frcmod
LLP = loadMol2 LLP.mol2

# Load protein structure
mol = loadPdb 1WDW_prepared.pdb

# Solvate: 10 Å buffer, cubic box
solvatebox mol TIP3PBOX 10.0

# Neutralize with counterions
addIonsRand mol Na+ 0
addIonsRand mol Cl- 0

# Check and save
check mol
saveAmberParm mol system.parm7 system.inpcrd

quit
EOF

tleap -f tleap.in > tleap.log 2>&1
```

### 3.3 搭建后验证


## 第四步：最小化（两步）

> **来源**：JACS 2019 SI p.S2 — "A two-stage geometry optimization approach was performed. The first stage minimizes the positions of solvent molecules and ions imposing positional restraints on solute by a harmonic potential with a force constant of 500 kcal mol⁻¹Å⁻², and the second stage is an unrestrained minimization of all the atoms in the simulation cell."

### 4.1 Stage 1: Restrained Minimization

```
min1: restrained minimization (solute fixed, 500 kcal/mol/A^2)
&cntrl
  imin=1,
  maxcyc=10000,
  ncyc=5000,
  ntb=1,
  ntr=1,
  restraint_wt=500.0,
  restraintmask='!@H= & !:WAT & !:Na+ & !:Cl-',
  cut=8.0,
  ntpr=500,
/
```

| 参数             | 值    | 含义                                 | 来源                             |
| ---------------- | ----- | ------------------------------------ | -------------------------------- |
| `restraint_wt` | 500.0 | 溶质约束力常数                       | SI p.S2 "500 kcal mol⁻¹Å⁻²" |
| `maxcyc`       | 10000 | 最大迭代步数                         | 工程选择                         |
| `ncyc`         | 5000  | 前 5000 步用最速下降，后面用共轭梯度 | 工程选择                         |

### 4.2 Stage 2: Unrestrained Minimization

```
min2: unrestrained minimization (all atoms free)
&cntrl
  imin=1,
  maxcyc=10000,
  ncyc=5000,
  ntb=1,
  ntr=0,
  cut=8.0,

```

## 第五步：阶梯加热（0 → 350 K，7 步）

> **来源**：JACS 2019 SI p.S2–S3 —
> "All systems were gently heated using seven 50 ps steps, incrementing the temperature 50 K each step (0-350 K) under constant-volume and periodic-boundary conditions. Water molecules are treated with the SHAKE algorithm such that the angle between the hydrogen atoms is kept fixed. Long-range electrostatic effects are modeled using the particle-mesh-Ewald method. An 8Å cutoff was applied to Lennard-Jones and electrostatic interactions. Decreasing harmonic restraints were applied to the protein (210, 165, 125, 85, 45, 10 kcal/mol Å²) during the thermal equilibration, with the Langevin scheme used to control and equalize the temperature. The time step is kept at 1 fs during the heating stages, allowing potential inhomogeneities to self-adjust."

### 5.1 加热参数总表

| Step  | 温度范围 (K) | 时长  | dt   | Restraint (kcal/mol·Å²) | 来源                 |
| ----- | ------------ | ----- | ---- | -------------------------- | -------------------- |
| heat1 | 0 → 50      | 50 ps | 1 fs | 210.0                      | SI p.S2-S3           |
| heat2 | 50 → 100    | 50 ps | 1 fs | 165.0                      | SI p.S2-S3           |
| heat3 | 100 → 150   | 50 ps | 1 fs | 125.0                      | SI p.S2-S3           |
| heat4 | 150 → 200   | 50 ps | 1 fs | 85.0                       | SI p.S2-S3           |
| heat5 | 200 → 250   | 50 ps | 1 fs | 45.0                       | SI p.S2-S3           |
| heat6 | 250 → 300   | 50 ps | 1 fs | 10.0                       | SI p.S2-S3           |
| heat7 | 300 → 350   | 50 ps | 1 fs | 10.0                       | **UNVERIFIED** |

> **⚠️ UNVERIFIED**：SI 列了 6 个 restraint 值 (210, 165, 125, 85, 45, 10)，但加热有 7 步。当前操作选择是 heat6 和 heat7 都使用 10.0。

### 5.2 heat1.in 模板（其他 step 改温度和 restraint）

```

&cntrl
  imin=0,
  irest=0,
  ntx=1,
  nstlim=50000,
  dt=0.001,
  ntc=2,
  ntf=2,
  ntt=3,
  gamma_ln=1.0,
  temp0=50.0,
  nmropt=1,
  ntb=1,
  ntp=0,
  cut=8.0,
  ntr=1,
  restraint_wt=210.0,
  restraintmask='!@H= & !:WAT & !:Na+',
  ntpr=5000,
  ntwx=5000,
  ntwe=5000,
  ntwr=50000,
  ioutfm=1,
/
&wt
  TYPE='TEMP0', ISTEP1=0, ISTEP2=50000, VALUE1=0.0, VALUE2=50.0,
/
&wt TYPE='END' /
```

| 参数             | 值                    | 含义                      | 来源                                     | 是否需要改     |
| ---------------- | --------------------- | ------------------------- | ---------------------------------------- | -------------- |
| `dt`           | 0.001 (1 fs)          | 加热阶段时间步长          | SI p.S3 "1 fs during the heating stages" | **不变** |
| `nstlim`       | 50000                 | 50000 × 0.001 ps = 50 ps | SI p.S2 "seven 50 ps steps"              | **不变** |
| `ntc=2, ntf=2` | SHAKE on bonds with H | SI p.S2 "SHAKE algorithm" | **不变**                           |                |
| `ntt=3`        | Langevin thermostat   | SI p.S3 "Langevin scheme" | **不变**                           |                |
| `cut=8.0`      | 8 Å cutoff           | SI p.S3 "8Å cutoff"      | **不变**                           |                |
| `ntb=1, ntp=0` | NVT (constant volume) | SI p.S2 "constant-volume" | **不变**                           |                |

## 第九步：Path CV 构建（15 帧线性插值）

> **来源**：JACS 2019 SI p.S3 —
> "a path of 15 conformations from an open (s(R) = 1) to closed state (s(R)=15), was generated. Guided by structural information we restricted the path of structures to the alpha carbons of the COMM domain (residues 97-184) and a region located at the base of COMM domain (residues 282-305), see Figure S1. The λ parameter was computed as 2.3 multiplied by the inverse of the mean square displacement between successive frames, 80."

### 9.1 路径构建参数

| 参数            | 值                     | 含义                                      | 来源                          |
| --------------- | ---------------------- | ----------------------------------------- | ----------------------------- |
| Open endpoint   | 1WDW (Cα)             | Open 参考结构                             | SI p.S2, Fig.S1               |
| Closed endpoint | 3CEP (Cα)             | Closed 参考结构                           | SI Fig.S1                     |
| 帧数            | 15                     | O(s=1) → C(s=15)                         | SI p.S3 "15 conformations"    |
| 原子选择        | Cα of 97-184, 282-305 | COMM domain + 基底                        | SI p.S3                       |
| λ              | 2.3 / MSD ≈ 0.029     | 路径帧间距归一化                          | SI p.S3 "λ...2.3...MSD...80" |
| 插值方法        | 笛卡尔线性插值         | 帧 i = Open + (i-1)/14 × (Closed - Open) | 标准 Path CV 构建             |

### 9.2 插值脚本

```python
import MDAnalysis as mda
import numpy as np

# 加载 Open 和 Closed
u_open = mda.Universe('1WDW_open.pdb')
u_closed = mda.Universe('3CEP_closed.pdb')

# 选择 COMM domain Cα
sel = 'name CA and (resid 97:184 or resid 282:305)'
ca_open = u_open.select_atoms(sel).positions
ca_closed = u_closed.select_atoms(sel).positions

# 线性插值 15 帧
for i in range(15):
    frac = i / 14.0
    coords = ca_open + frac * (ca_closed - ca_open)
    # 保存为 PDB frame (frame_01.pdb ... frame_15.pdb)

# 计算 MSD
diffs = [np.mean(np.sum((frames[i+1] - frames[i])**2, axis=1))
         for i in range(14)]
mean_msd = np.mean(diffs)
lam = 2.3 / mean_msd
print(f"MSD = {mean_msd:.3f} Å², λ = {lam:.6f} Å⁻²")
# 期望: MSD ≈ 80, λ ≈ 0.029
```

> **当前 lambda 张力**：文献目标 λ = 0.029 (MSD = 80)，本地路径计算给出 λ = 0.033910 (MSD = 67.826)。差异可能来自路径构建细节（对齐方式、残基选择）。需在执行时确认。

> "After an initial metadynamics run, we extracted ten snapshots for each system covering approximately all the conformational space available. ## 第十二步：FES 重建

> **来源**：JACS 2019 SI p.S4 —
> "The free energy landscape associated with the metadynamics CVs is estimated by summing the Gaussian potentials deposited by all walker replicas as a function of the CVs values."
> "The local energy minima and the associated representative structures were labeled as open (O), partially closed (PC) and closed (C) accordingly with the s(R) CV values; (O)=1-5, (PC)=5-10 and (C)=10-15."

### 12.1 合并 HILLS 并重建 FES

```bash
# 合并所有 walker 的 HILLS 文件
cat walker_*/HILLS > HILLS_all

# 重建 FES
plumed sum_hills --hills HILLS_all \
                 --outfile fes.dat \
                 --mintozero \
                 --kt 0.695
```

| 参数            | 值    | 含义                   |
| --------------- | ----- | ---------------------- |
| `--kt`        | 0.695 | kT at 350 K (kcal/mol) |
| `--mintozero` | —    | 将 FES 最低点设为 0    |

> **⚠️ FP-关键**：如果漏掉 `--kt`，basin 位置不变但所有 ΔG 和 barrier 数值会系统性偏差。

### 12.2 Basin 标记

| s(R) 区间 | 构象标记              | 来源                |
| --------- | --------------------- | ------------------- |
| 1-5       | Open (O)              | SI p.S4 "(O)=1-5"   |
| 5-10      | Partially Closed (PC) | SI p.S4 "(PC)=5-10" |
| 10-15     | Closed (C)            | SI p.S4 "(C)=10-15" |

### 12.3 收敛检查

> **来源**：SI p.S4 — "The convergence of the recovered FEL was evaluated by monitoring the energy difference between selected regions of the conformational surface along simulation time."

```bash
# Blockwise FES 重建（每 10 ns 一块）
for t in 10 20 30 40 50; do
    plumed sum_hills --hills HILLS_all \
                     --outfile fes_${t}ns.dat \
                     --mintozero --kt 0.695 \
                     --stride 5000 \
                     --max_time ${t}000
done

# 跟踪 ΔG(O-C) 随时间变化
# 如果最后几个 block 的 ΔG 不再显著漂移 → 收敛
```
