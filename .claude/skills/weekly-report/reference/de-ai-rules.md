# De-AI Language Rules

These rules are mandatory. Apply them during writing (Step 3) and verify
during validation (Step 5).

## Banned phrases

Replace on sight. No exceptions.

| Don't write | Write instead |
|-------------|---------------|
| delve into | read, looked into, dug into |
| leverage | use, apply |
| utilize | use |
| comprehensive | be specific: "covered X, Y, Z" |
| robust | works, reliable, stable |
| significant | use numbers: "3 kcal/mol difference" |
| fundamental | basic, key |
| multifaceted | list the facets |
| intricate | complex, involved |
| it's important to note | [just state the fact] |
| this is particularly interesting because | This matters because |
| moving forward | next, later, or delete entirely |
| in terms of | rephrase without it |

## Banned transitions

These are filler. Start a fresh sentence or combine with the previous one.

- Additionally
- Furthermore
- Moreover
- Notably
- Specifically
- Importantly
- It is worth noting that

## Proper nouns

- Anima Lab (capital A)
- PLUMED (all caps)
- MetaDynamics (capital M, capital D)
- GROMACS (all caps)
- AMBER (all caps)

## First-use term explanation

The first time a technical term appears in the report, add a brief
parenthetical explanation. After the first use, use the bare term.

Examples:
- "MetaDynamics (an enhanced sampling method that accelerates rare conformational transitions)"
- "PLUMED (a plugin that adds MetaDynamics capabilities to molecular dynamics engines)"
- "FES (free energy surface: a map showing which protein conformations are energetically preferred)"
- "COMM domain (a ~88-residue region of TrpB whose open/closed state controls catalytic activity)"

## Punctuation

No em dashes (---, —, –) anywhere. If a sentence needs an em dash, rewrite it
as two sentences or use a colon. This applies to the markdown draft and the
final .docx output.

## Sentence-level rules

1. **One comma max per sentence.** Two or more commas = split into two sentences.
2. **Use "I" not "we."** Only "we" if genuinely co-authored with another person.
3. **Numbers over abstractions.** "read 4 papers" not "conducted a literature review."
4. **Direct on facts.** "The 0B2 variant carries six mutations." Not "one would observe six mutations."
5. **Honest about uncertainty.** "I don't understand this yet" not "further investigation is warranted."

## Structural-level rules (harder to catch, equally important)

These come from research on AI detection (Kujur 2025). Perplexity and
repetitive structure are the primary tells.

1. **Vary paragraph length aggressively.** If three paragraphs in a row are
   similar length (say, all 4 sentences), break one into 2 sentences or
   combine two into a longer block.
2. **Break symmetry.** If Section 2 has three subsections and each follows the
   pattern "topic sentence, 2 insights, plan implication" -- change the third
   one. Maybe start with the plan implication, or skip it entirely.
3. **Allow imperfection.** Real writing has hedges. Include at least one genuine
   uncertainty per major section ("I'm not 100% sure this is right, but...").
4. **Don't over-signpost.** Remove sentences that announce what you're about to say.
   Just say it. "In this section I discuss X" = delete.
5. **Vary sentence openers.** Don't start three consecutive sentences with "The"
   or "I". Restructure one.

## Attribution rules (critical)

The report must never mention AI tools. All work is "I did X."

| Don't write | Write instead |
|-------------|---------------|
| I used Claude Code to install GROMACS | I installed GROMACS 2026.0 with PLUMED support via conda-forge |
| AI-assisted extraction of parameters from SI | I extracted all MetaDynamics parameters from the JACS 2019 SI |
| The 6-stage pipeline is designed for AI agents | I organized the replication into 6 modular stages based on Xu & Yang 2026 |
| Claude read the paper and generated notes | I read the paper and wrote structured notes |
| The manifest was auto-generated by the Profiler | I created a frozen manifest defining all simulation parameters and success criteria |
| SSH ControlMaster was configured for AI access | I set up persistent SSH connections for efficient remote work |

**General principle:**
- Installations, configurations, extractions, analyses: "I did X"
- Tools and methods: describe the tool itself ("using conda-forge"), never the AI
- Pipeline/manifest/skills: "modular workflow", "frozen protocol", "reusable templates"
- If automation must be mentioned: "scripted" or "automated", never "AI-assisted"
