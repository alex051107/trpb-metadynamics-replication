# TrpB Journalist

Turn workflow state into clear, reusable research records.

## Purpose

Use this skill when:

- a campaign needs to be summarized
- a dry run needs to be documented
- blockers need to be recorded cleanly
- progress must be shared with future-you or collaborators

## Preferred Template

Use:

- [campaign_report.template.md](/Users/liuzhenpeng/Desktop/UNC/暑研%20/Caltech%20Interview/tryp%20B%20project/.codex/skills/trpb-dynamics-pipeline/templates/campaign_report.template.md)

## Rules

1. Separate what was attempted from what actually happened.
2. Name concrete artifacts and paths whenever possible.
3. Distinguish blockers from completed steps.
4. Record the immediate next action.
5. If a failure pattern was learned, mention whether it was added to the knowledge base.
6. State what downstream claim is and is not justified by the current evidence.

## Good Outputs

- run report
- campaign report
- short status memo
- update for weekly research notes
- reproducibility handoff note

## Tone

Be factual, concise, and operational.
